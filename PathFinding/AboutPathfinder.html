Algorithms

At the top of the application, there are several different algorithms that are either used for pathfinding (finding a path from the start node to the end node) or used for generating mazes.

  • Dijkstra's Algorithm - Pathfinding
  • A* - Pathfinding
  • Breadth-First Search - Pathfinding
  • Depth-First Search - Pathfinding
  • Bidirectional Search - Pathfinding
  • Prim's Maze Generation - Maze Generation
  • Recursive Maze Generation - Maze Generation

Nodes

On the grid, the pink circular node is the start node and the blue circular node is the end node.

When a pathfinding algorithm is executed, the square-shaped grid nodes that the algorithm has visited will change to a light-purple color to showcase how the algorithm works, while the unvisited nodes remain white. Since wall nodes are impassable, they will remain dark-purple during the algorithm. If a path from the start node to the end node exists, a light blue path will be drawn at the conclusion of the algorithm. Otherwise, no path will be drawn.

When a maze generation algorithm is executed, walls will be added, so that pathfinding algorithms must go around them when looking for a valid path.

The user is also able to add walls or erase walls by checking the Draw Wall checkbox or Erase Wall checkbox, respectively, and dragging their mouse across the grid.

Other Features

Several other features are in this application.

  • Clear Board - Clears all animations and walls
  • Speed Slider - Changes the animation speed
  • Draw Wall - If this box is checked, the user can draw walls on the board by dragging the mouse across the grid
  • Erase Wall - If this box is checked, the user can erase walls on the board by dragging the mouse across the grid
  • Show Steps - If this box is checked, the animation for the algorithm will be showed; otherwise, only the final product will be visible
  • Finish Algorithm - If an algorithm is occurring, this button can be pressed to immediately finish it and show the final product
